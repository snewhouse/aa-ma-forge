## Fresh-Agent Simulation Spot-Checks

Simulated fresh-agent implementations of three high-risk steps to verify two independent agents would produce equivalent output.

### Step 1.2 (SQLite schema DDL)
- **Pre-v2:** ambiguous — table definitions stated in prose, FK/index details implicit.
- **Post-v2:** DDL pinned verbatim in plan; two agents would produce identical `schema.sql`. **Passes simulation.**

### Step 2.3 (WAL replay logic)
- **Pre-v3:** ambiguous — "replay idempotently" left state machine unspecified. Two simulated agents diverged at the crash-after-commit-before-ack case (one would double-apply, one would skip-without-ack).
- **Post-v3:** replay state diagram embedded verbatim; idempotency key `(op, prev_user_version, content_sha)` pinned. Two agents would produce identical semantics. **Passes simulation.**

### Step 3.7 (`aa_ma_context` extraction)
- **Pre-v3:** ambiguous — "extract files and symbols from task artifacts" with no regex or filtering rule. Three simulated agents produced three different extractors (naive grep, import re + stat, hallucinated yaml frontmatter parser).
- **Post-v3:** file-mention and symbol-mention regexes pinned; filesystem existence check for files; `symbols.name` lookup for symbols; golden fixture `tests/fixtures/aa_ma_context/sample-task/` with `expected.json` for byte-for-byte test. **Passes simulation.**
