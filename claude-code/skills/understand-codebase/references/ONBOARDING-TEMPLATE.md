# ONBOARDING-TEMPLATE — the root `ONBOARDING.md` skeleton

This is the output contract for `<repo>/ONBOARDING.md`. The synthesizer fills it. Rules:

- **Quick tier** writes only the "10-minute orientation" section + the Provenance block + a footer
  pointing at `--standard`. Stop there.
- **Standard / Deep** write the whole thing; each major section is a *summary* with a link to its
  `.claude/onboarding/NN-*.md` deep-dive for the full version.
- Every claim cites evidence (`file:line` / command / git fact / count) or says `not found — gap`.
- Never include a secret value. `.env*` → variable *names* only.
- It must be readable top-to-bottom by a human who has never seen the repo. Plain language.
- It works with the `ShareOnboardingGuide` tool (it lives at the repo root, named `ONBOARDING.md`).
- Idempotent: a re-run updates this file in place and refreshes the Provenance block.
